## The methodology gap (the real finding)

The headline **98.5% accuracy / 0% hallucination** in `COMPLETE_EXPERIMENT_RESULTS.md`
came from **manual, human-scored** evaluation on a curated document set. This
automated run reports **~11%** accuracy and ~12 "hallucinations" per doc on the
**same model doing the same task well**. The entire gap is the *evaluator*, not
the model:

1. **Key-alignment, not content.** `ExtractionEvaluator._evaluate_dict` only
   credits a field when the **same dict key** holds matching values in both the
   model output and the FUNSD ground truth. Claude returns
   `{header, questions, answers, other}`; FUNSD's schema differs — so correct
   extractions don't line up by key and score as misses. (Even clean, level-0
   docs score ~9.7% — a tell that the metric, not the input, is the ceiling.)
2. **Over-eager hallucination flag.** Any extracted span not found verbatim in
   the ground-truth field values is flagged as a hallucination. Claude correctly
   extracts page text outside FUNSD's narrow field set, so honest extractions get
   counted as fabrications — inflating the count to ~12/doc.

**Takeaway:** the automated numbers measure the brittleness of a naive
exact-key-match metric, not Claude's capability. The capability signal is in the
manual eval and in the valid-JSON rate (20/20). *Designing the metric is the hard
part of the eval* — a stricter automated metric than the task warrants will
report failure on correct work.

## What still holds regardless of the metric

- **Relative chaos degradation is real.** Even on the suppressed automated scale,
  accuracy falls monotonically (9.7% → 0%) as degradation increases, collapsing
  around **level 2–3** — consistent with the manual finding that blur/noise past
  ~60% degradation breaks extraction. The *curve* is meaningful even when the
  *absolute* numbers are metric-limited.
- **JSON reliability is high** (20/20 valid) — the model returns parseable,
  structured output under load.

## Open follow-up

The honest next step is to fix the automated evaluator so it reflects extraction
quality: align Claude's output structure to the FUNSD schema before comparison,
and scope hallucination detection to the relevant field set. Until then, cite the
**manual** result for capability and **this** result for the eval-design lesson.
